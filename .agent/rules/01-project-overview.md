# 01: プロジェクト概要

## プロダクト名

**PdM Assistant**（壁打ちパートナー）

## ミッション

要件定義の解像度を上げるAI壁打ちツール。

ユーザー（PdM/企画者）の「Why（なぜやるのか）」の思考を深め、ステークホルダーが腹落ちする水準まで磨き上げる。

## 根本思想

> 人間もAIも、情報のやり取りでは発信者の意図の50%程度しか正確に伝わらない。
> 残りの50%は受け手が「空気を読んで」補完して理解している。
> この「補完に頼る50%」を減らし、大事なことを明文化するのが要件定義の役割。

## コア機能

1. **5観点分析**: 背景・課題・ターゲット・効果・緊急性で入力を構造化
2. **壁打ち対話**: ソクラテス式問答で思考を深化
3. **コンタミ検知**: How/What混入を検知し、Whyに引き戻す
4. **横展開**: 1つの発言から関連する他の観点を自動更新
5. **観点チェック**: 5観点の網羅性と品質を診断

## 技術スタック

- **フロントエンド**: Vanilla HTML/CSS/JS（フレームワークなし）
- **AI**: OpenAI API（gpt-4o-mini）
- **DB**: Supabase（PostgreSQL）
- **ホスティング**: Vercel（Edge Functions + 静的配信）
- **ローカル開発**: Python `http.server`

## 重要な設計原則

- **開発AIルール ≠ プロダクトAI設計**: `.agent/rules/`は開発プロセス、`src/ai/`はプロダクトAIの定義
- **Flows/Crews分離**: 決定論的制御（Flow層）と知的処理（Crew層）を分離
- **バックアップ戦略**: 構造変更時は元ファイルを残し、新構造の動作確認後に削除
