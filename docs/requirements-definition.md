# PdMアシスタントAIエージェント 要件定義

## 概要

PdM（プロダクトマネージャー）、PM（プロジェクトマネージャー）、ディレクターなどの職種が、高品質な要件定義書を作成するための対話型AIアシスタントです。

特に**「Why（なぜ作るのか）」**と**「What（何を作るのか）」**の深掘りと構造化に特化しており、ユーザーが安易に「How（どう作るか）」に飛びつくことを防ぎます。

ユーザーが入力した曖昧なアイデアや目的に対し、AIが徹底的に「なぜ？」を問い直し、壁打ち相手となることで、思考の言語化と解像度の向上をサポートします。未経験者でも、このエージェントとの対話を通じて、本質的な価値定義に基づいた要件定義を作成できるようになることを指針とします。

## なぜ作るのか

### 1. 「How」偏重による「質の低いアウトプット」の解決
多くの現場において、課題の本質（Why）や提供価値（What）が十分に整理されないまま、解決策（How）の議論や実装に進んでしまうケースが散見されます。
AIツールを活用しても、入力されるプロンプト（指示）の質が低ければ、出力される回答も「指示通りだが、本質的な課題解決になっていない」ものになります。
AIは文脈を補完して尤もらしい回答を生成するのが得意なため、ユーザー自身が思考の浅さに気づきにくいという課題があります。
このエージェントは、あえて「How」の議論を止め、「Why」と「What」が腹落ちするまで徹底的に詰め切るプロセスを強制することで、真に質の高い要件定義を実現します。

### 2. 言語化と構造化のサポート（思考の壁打ち）
「なんとなく作りたいものはあるが、言葉にできない」「背景まで含めて正しく伝えられない」というユーザーの課題に対し、AIが多角的な視点から問いかけを行います。
単なるチャットボットではなく、必要な観点をAI側から提示し、ユーザーに回答させることで、思考の整理と言語化を促します。
「Why」の中に「How」や「What」が混ざっている場合などはそれを指摘し、論理的な整合性の取れた「完璧なWhy」を構築するまでサポートします。

### 3. 教育・育成コストの削減とスキルの底上げ
ジュニアメンバーの教育には多大なコストがかかり、また要件定義のフィードバック（FB）は属人的になりがちで難しい側面があります。
このエージェントは、熟練のPdMが壁打ち相手となるかのように、適切な問いかけとフィードバックを提供します。
ユーザーはエージェントとの対話を通じて、要件定義に必要な思考プロセスや視座を疑似体験し、実践の中で「整理する癖」や「正しい粒度での言語化」を学ぶことができます。

### 4. ステークホルダーとの合意形成の円滑化
解像度の高い「Why」を事前に構築することで、チームメンバーやステークホルダーとの認識齟齬を減らします。
「なぜやるのか」が明確であれば、その後の「What」や「How」の議論もスムーズに進み、手戻りのリスクを最小限に抑えることができます。

## What（何を作るのか）

### 1. 機能要件

#### 1.1 Whyの深掘りと完璧なWhyの構築
*   **深掘り質問生成**: ユーザーが入力した「Why」に対し、不足している観点や論理的飛躍を指摘する質問を生成します。
    *   例：「ターゲット選定の根拠は？」「競合優位性は具体的に何か？」「なぜ今やる必要があるのか？」
*   **How/Whatの混入検知**: ユーザーの回答に解決策（How）や成果物（What）が含まれている場合、それを指摘し、「なぜそれが必要なのか」という本質的な動機（Why）に引き戻します。
*   **合意形成支援**: 生成されたWhyが「完璧なWhy」であるかをユーザーとエージェントで合意形成します。
    *   「完璧なWhy」の定義：ステークホルダーが納得し、論理的矛盾がなく、その後のWhat/Howの指針となるレベルまで言語化されている状態。

#### 1.2 Whatへの移行プロセス
*   **移行基準**: ユーザーとエージェント双方が「Why」の内容に合意し、かつステークホルダー（仮想的なロールプレイを含む）からの反論にも耐えうると判断された場合にのみ、次のステップ（What）へ移行します。
*   **What定義サポート**: 確定したWhyに基づき、「どのような解決策（What）」を提供すべきかを定義します。ここでも安易なHow（技術選定や詳細仕様）への飛びつきを防ぎます。

#### 1.3 方針の壁打ち・選定（Why this approach）
*   **複数案の比較検討**: 単一の解決策に飛びつくのではなく、複数の実現方針（オプション）を検討させます。
*   **選定理由の言語化**: なぜその方針を選んだのか、メリット・デメリット・リスクの観点から合理的理由を言語化させます。これが「完璧なWhat」の一部となります。

### 2. 体験設計（UX）

#### 2.1 入力インターフェース
*   **構造化された入力フォーム**: 自由記述のチャット欄だけでなく、AIが提示した観点（例：背景、課題、ターゲット、提供価値）ごとに個別の入力フィールドを用意します。これにより、ユーザーは思考を整理しやすくなります。
*   **ガイド付き入力**: 各フィールドには、「ここでは〇〇について記述してください」といった具体的なガイドや例文を表示します。

#### 2.2 インラインでの対話と解像度の可視化
*   **インラインフィードバック**: ユーザーの回答に対し、チャット形式で流れてしまうのではなく、該当箇所にインラインでコメントや指摘を表示します（Google Docsのコメント機能のようなイメージ）。
*   **解像度の履歴保持**: 初期の曖昧な記述から、AIとの対話を経てどのように洗練（ブラッシュアップ）されたかの履歴を保持し、可視化します。「思考が磨かれるプロセス」自体をユーザーが体験・学習できるようにします。
